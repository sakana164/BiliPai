# 顶部指示器形变对齐设计

## 目标

- 顶部分类指示器的基础横向放大倍数与底部 Dock 完全一致。
- 顶部左右滑动时复用底部 Dock 的方向速度拉伸和纵向压缩。

## 实现

- 保持 `resolveBottomBarIndicatorLayerTransform` 作为唯一形变算法，不新增顶部专用倍率。
- 修正顶部 Pager 滑动的状态输入，使滑动期间完整进入底栏拖拽形变状态。
- 顶部直接拖拽继续使用现有拖拽状态，Pager 滑动使用稳定的位移速度输入。
- 不修改底栏常量、底栏视觉和其他分段控件。

## 验证

- 纯策略测试验证顶部静态、拖拽和正反方向速度形变与底栏结果一致。
- 运行顶部指示器相关单测和 `:app:compileDebugKotlin`。
- 设备验证顶部“推荐—游戏”左右滑动时，胶囊放大幅度及方向拉伸与底栏一致。

## 风险

- 顶部指示器高度较小，相同纵向压缩会比底栏更明显；本次按“完全对齐”要求不增加顶部修正系数。
